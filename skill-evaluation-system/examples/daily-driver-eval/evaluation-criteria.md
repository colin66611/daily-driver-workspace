# Daily Driver Skill 评估标准

## 一、评分体系

### 1.1 功能性评分 (权重 40%)

| 等级 | 描述 | 分数范围 |
|---|---|---|
| **优秀** | 所有功能完美执行,输出准确无误 | 90-100分 |
| **良好** | 核心功能正常,次要功能有小瑕疵 | 75-89分 |
| **合格** | 核心功能基本正常,有已知限制 | 60-74分 |
| **不合格** | 核心功能无法正常执行 | 0-59分 |

**评分细则**:
- 每个功能模块独立评分
- 核心模块(阶段1-3,阶段5-6)权重更高
- 文件创建准确性加倍权重

---

### 1.2 流程完整性评分 (权重 25%)

| 等级 | 描述 | 分数范围 |
|---|---|---|
| **流畅** | 流程无卡顿,用户指引清晰,体验优秀 | 90-100分 |
| **顺畅** | 流程基本流畅,偶有轻微确认 | 75-89分 |
| **可用** | 流程完整但需多次确认,体验一般 | 60-74分 |
| **断裂** | 流程中断或无法完成闭环 | 0-59分 |

**评分细则**:
- 用户确认次数(越少越好)
- 流程中断次数(应为0)
- 错误处理流畅度
- 用户指引清晰度

---

### 1.3 文件可靠性评分 (权重 20%)

| 等级 | 描述 | 分数范围 |
|---|---|---|
| **完全可靠** | 所有文件操作准确,内容完整规范 | 90-100分 |
| **基本可靠** | 文件操作正常,内容有小瑕疵 | 75-89分 |
| **部分可靠** | 部分文件操作失败或内容不完整 | 60-74分 |
| **不可靠** | 文件操作频繁失败或内容错误 | 0-59分 |

**评分细则**:
- 文件创建成功率
- 文件内容完整性
- 文件命名规范性
- 文件更新及时性

---

### 1.4 边界处理评分 (权重 10%)

| 等级 | 描述 | 分数范围 |
|---|---|---|
| **鲁棒** | 所有边界条件正确处理,有友好提示 | 90-100分 |
| **健壮** | 主要边界条件处理正常,有提示 | 75-89分 |
| **脆弱** | 部分边界条件处理不当 | 60-74分 |
| **崩溃** | 边界条件导致崩溃或无响应 | 0-59分 |

**评分细则**:
- 异常检测能力
- 错误提示友好度
- 恢复机制完整性
- 不中断运行能力

---

### 1.5 触发准确性评分 (权重 5%)

| 等级 | 描述 | 分数范围 |
|---|---|---|
| **精准** | 触发率 100%,误触发率 0% | 90-100分 |
| **准确** | 触发率 >90%,误触发率 <10% | 75-89分 |
| **一般** | 触发率 >70%,误触发率 <30% | 60-74分 |
| **混乱** | 触发率过低或误触发过多 | 0-59分 |

**评分细则**:
- 应触发场景触发率
- 不应触发场景误触发率
- 模糊场景处理能力

---

## 二、总体评分标准

### 2.1 综合评分计算
```
总分 = 功能性 × 0.40 
     + 流程完整性 × 0.25 
     + 文件可靠性 × 0.20 
     + 边界处理 × 0.10 
     + 触发准确性 × 0.05
```

### 2.2 总体评级

| 总分 | 评级 | 状态 | 建议 |
|---|---|---|---|
| **90-100** | **生产级** | 可直接使用 | 继续优化细节 |
| **75-89** | **可用级** | 基本可用 | 改进已知问题 |
| **60-74** | **测试级** | 需要测试完善 | 修复核心问题 |
| **0-59** | **开发级** | 不可用 | 重新开发核心功能 |

---

## 三、通过标准

### 3.1 最低通过标准
- 功能性评分 ≥ 60分
- 流程完整性评分 ≥ 60分
- 文件可靠性评分 ≥ 75分
- 总分 ≥ 70分

### 3.2 推荐使用标准
- 功能性评分 ≥ 75分
- 流程完整性评分 ≥ 75分
- 文件可靠性评分 ≥ 90分
- 边界处理评分 ≥ 75分
- 总分 ≥ 80分

### 3.3 生产发布标准
- 所有单项评分 ≥ 85分
- 总分 ≥ 90分
- 无关键功能失败
- 边界条件全部处理

## 四、关键检查点

### 4.1 必须通过的检查点 (Critical)
以下检查点必须 100% 通过,否则视为不合格:

1. ✅ **文件创建**: task_plan.md, findings.md, progress.md 必须正确创建
2. ✅ **文件内容**: 三文件必须包含所有必需字段
3. ✅ **状态更新**: 任务完成时状态必须正确更新
4. ✅ **流程闭环**: 从启动到收尾必须完整闭环
5. ✅ **不崩溃**: 任何场景都不应导致崩溃

### 4.2 重要检查点 (Important)
以下检查点至少 90% 通过:

1. ⚠️ **任务延续**: 昨天未完成任务正确检测和延续
2. ⚠️ **实时更新**: 执行过程中文件实时更新
3. ⚠️ **并行执行**: 多任务并行管理
4. ⚠️ **用户指引**: 每阶段用户指引清晰
5. ⚠️ **错误提示**: 异常场景友好提示

### 4.3 次要检查点 (Minor)
以下检查点至少 70% 通过:

1. 📝 **间隔处理**: 多天间隔后的使用处理
2. 📝 **取消处理**: 用户中途取消的处理
3. 📝 **权限处理**: 文件权限问题处理
4. 📝 **触发准确**: 模糊场景触发处理
5. 📝 **性能表现**: 文件操作响应时间

---

## 五、失败判定标准

### 5.1 立即失败条件
出现以下任一情况,立即判定为失败:

1. ❌ 文件创建失败导致无法继续
2. ❌ 流程中断无法完成闭环
3. ❌ 关键数据丢失或错误
4. ❌ 系统崩溃或无响应
5. ❌ 用户无法理解下一步操作

### 5.2 需改进条件
出现以下情况需要改进:

1. ⚠️ 文件内容不完整或格式错误
2. ⚠️ 用户需要多次询问才能理解
3. ⚠️ 边界条件处理不当但有提示
4. ⚠️ 触发不准确但可手动调用
5. ⚠️ 性能慢但功能正常
