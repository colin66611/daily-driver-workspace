# Daily Driver Skill 评估测试方案

## 一、评估目标

### 1.1 核心目标
评估 daily-driver skill 是否能够:
1. **正确触发** - 在合适的场景下被正确激活
2. **功能完整** - 所有声明功能都能正常执行
3. **流程顺畅** - 工作流程各阶段衔接自然
4. **文件驱动** - 实时创建和更新文件作为外部记忆
5. **强制闭环** - 确保任务完整闭环,不悬空对话

### 1.2 评估维度
- **功能性**: 每个阶段功能是否正常工作
- **可靠性**: 文件操作是否可靠,数据是否准确
- **可用性**: 用户交互是否清晰,指令是否明确
- **完整性**: 工作流是否完整覆盖启动到收尾
- **鲁棒性**: 边界条件和异常情况处理

## 二、测试范围

### 2.1 核心功能模块
1. **阶段 0.5**: 检测昨天未完成任务
   - 昨天目录查找
   - 任务状态读取
   - 用户询问流程
   - 任务延续执行

2. **阶段 1**: 今日文件夹创建
   - 今日状态检查
   - 文件夹创建
   - 初始化文件生成

3. **阶段 2**: 任务确认
   - 并行执行处理
   - 首次初始化流程
   - 用户意图理解

4. **阶段 3**: 任务文件夹创建
   - 任务目录创建
   - 三文件模板生成
   - 每日记录更新

5. **阶段 4**: 执行过程实时更新
   - progress.md 实时追加
   - findings.md 发现记录
   - task_plan.md 状态更新

6. **阶段 5**: 任务完成标记
   - 完成状态更新
   - 每日记录同步
   - 完成记录生成

7. **阶段 6**: 收尾归档
   - 未完成任务检查
   - 完成报告生成
   - 任务归档执行

### 2.2 边界条件
- 连续多天未使用
- 跨周末/假期使用
- 并行任务执行
- 任务中途中断
- 文件权限问题
- 路径不存在情况

## 三、测试用例分类

### 3.1 功能性测试 (Functional Tests)
测试核心功能是否正常工作,每个功能模块至少 3 个测试用例。

### 3.2 流程完整性测试 (Workflow Tests)
测试完整工作流从启动到收尾的流畅性,包括:
- 首次使用流程
- 连续使用流程
- 中断后恢复流程

### 3.3 边界条件测试 (Edge Case Tests)
测试特殊场景和异常情况的处理能力。

### 3.4 文件系统测试 (File System Tests)
验证文件创建、更新、读取的准确性和可靠性。

### 3.5 触发准确性测试 (Trigger Tests)
验证 skill 在不同场景下的触发准确性。

## 四、评估标准

### 4.1 功能正确性
- **通过**: 功能按预期执行,输出正确
- **部分通过**: 功能执行但有偏差或警告
- **失败**: 功能无法执行或输出错误

### 4.2 流程完整性
- **优秀**: 流程顺畅,无卡顿,用户指引清晰
- **良好**: 流程基本顺畅,有轻微卡顿
- **一般**: 流程有卡顿,需用户多次确认
- **失败**: 流程中断或无法完成

### 4.3 文件可靠性
- **可靠**: 文件创建/更新准确,内容完整
- **部分可靠**: 文件操作成功但内容有偏差
- **不可靠**: 文件操作失败或内容错误

### 4.4 触发准确性
- **精确触发**: 应触发场景 100% 触发,不应触发场景 0% 触发
- **正常触发**: 应触发场景 >80%,不应触发场景 <20%
- **触发异常**: 触发率过低或误触发过多

## 六、评估流程

### 6.1 准备阶段
1. 创建测试环境
2. 准备测试数据
3. 确认测试用例

### 6.2 执行阶段
1. 运行功能性测试
2. 运行流程测试
3. 运行边界测试
4. 运行文件系统测试
5. 运行触发测试

### 6.3 分析阶段
1. 收集测试结果
2. 分析失败原因
3. 识别改进点
4. 生成评估报告

### 6.4 迭代阶段
1. 根据评估结果改进 skill
2. 重新测试验证改进
3. 直到满足标准
