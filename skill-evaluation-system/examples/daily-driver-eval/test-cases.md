# Daily Driver Skill 测试用例集合

## 一、功能性测试用例

### TC-F01: 检测昨天未完成任务 - 正常流程
**测试目标**: 验证能否正确检测昨天的未完成任务

**测试场景**:
- 昨天有 2 个任务,其中 1 个完成,1 个进行中
- 今天文件夹不存在
- 用户未说"并行"

**输入**:
```
开始今天的工作
```

**预期输出**:
1. 检测到昨天的未完成任务
2. 显示任务列表和状态
3. 询问用户是否延续
4. 提供延续选项

**验证点**:
- [ ] 正确识别昨天日期
- [ ] 正确读取任务状态
- [ ] 任务列表显示完整
- [ ] 询问选项清晰明确

---

### TC-F02: 创建今日文件夹 - 首次使用
**测试目标**: 验证首次使用时今日文件夹创建

**测试场景**:
- 今天文件夹不存在
- 昨天没有未完成任务

**输入**:
```
开启今天的工作,我要开始写一个新的技术文档
```

**预期输出**:
1. 创建今日文件夹 `2026-04-03/`
2. 创建每日记录文件 `2026-04-03.md`
3. 创建想法停车场 `_Idea-Parking-Lot.md`(如不存在)
4. 询问任务详情

**验证点**:
- [ ] 文件夹名称正确(YYYY-MM-DD)
- [ ] 每日记录文件创建
- [ ] 文件内容模板正确
- [ ] 用户询问流程正常

---

### TC-F03: 创建任务文件夹 - 标准任务
**测试目标**: 验证任务文件夹和三文件的创建

**测试场景**:
- 用户确认任务: M1 - 写技术文档

**输入**:
```
我要写一个关于 daily-driver skill 的技术文档,目标是让其他开发者了解如何使用这个工具
```

**预期输出**:
1. 创建 `task_M1_daily-driver技术文档/` 文件夹
2. 创建 `task_plan.md`
3. 创建 `findings.md`
4. 创建 `progress.md`
5. 在每日记录中添加任务条目

**验证点**:
- [ ] 文件夹命名规范(task_M1_简名)
- [ ] task_plan.md 内容完整(目标、交付物、标准)
- [ ] findings.md 模板正确
- [ ] progress.md 有初始化记录
- [ ] 每日记录任务条目正确

## 二、流程完整性测试用例

### TC-W01: 首次使用完整流程
**测试目标**: 验证从第一次使用到任务完成的完整流程

**测试场景**:
- 用户第一次使用 daily-driver
- 无历史数据

**完整流程**:
```
用户: 开始今天的工作
     → 系统创建今日文件夹
     → 询问今日任务
     
用户: 我要学习 Tauri 框架,目标是掌握基本用法
     → 系统创建 M1 任务文件夹
     → 生成三文件
     
用户: 展开 M1
     → task-worker 开始执行
     → 实时更新 progress.md
     
用户: 完成
     → 系统标记任务完成
     → 更新状态
     
用户: 结束今天的工作
     → 系统归档任务
     → 生成报告
```

**验证点**:
- [ ] 流程无中断
- [ ] 每阶段文件正确创建
- [ ] 用户指引清晰
- [ ] 最终状态正确

---

### TC-W02: 连续使用流程
**测试目标**: 验证连续多天使用的流程

**测试场景**:
- Day 1: 创建任务 M1,M2,完成 M1
- Day 2: 延续 M2,创建新任务 M3

**Day 1 流程**:
```
开始今天的工作
创建 M1: 学习 Tauri (完成)
创建 M2: 写文档 (进行中)
结束今天的工作
```

**Day 2 流程**:
```
开始今天的工作
→ 检测到昨天的 M2 未完成
→ 询问是否延续
→ 延续 M2 到今天
→ 创建 M3: 新任务
```

**验证点**:
- [ ] 昨天任务正确检测
- [ ] 延续流程顺畅
- [ ] 任务文件正确复制/链接
- [ ] 新任务创建正常

---

### TC-W03: 并行执行流程
**测试目标**: 验证并行执行场景

**测试场景**:
- 今天文件夹已存在
- 有任务 M1 进行中
- 用户说"并行"并添加新任务

**输入**:
```
并行开始新任务 M2
```

**预期输出**:
1. 不询问昨天任务延续
2. 直接显示今天现有任务
3. 创建新任务 M2
4. 保持 M1 继续执行

**验证点**:
- [ ] 不触发昨天任务检测
- [ ] 现有任务显示清晰
- [ ] 新任务创建成功
- [ ] 多任务并行管理

## 五、触发准确性测试用例

**测试目标**: 验证 skill 在应触发场景下正确触发

**验证点**:

**验证点**:

**测试目标**: 验证模糊场景的处理

- [ ] 模糊场景有确认机制

## 六、综合场景测试用例

### TC-C01: 完整一日工作流
**测试目标**: 验证完整一日的真实工作流程

**场景描述**:
```
09:00 - 开始今天的工作
09:05 - 创建 M1: 写技术文档
09:10 - 创建 M2: 学习新技术
09:15 - 展开 M1
10:30 - M1 完成第一阶段
11:00 - 发现关键信息,记录到 findings
12:00 - 完成 M1
14:00 - 展开 M2
16:00 - 完成 M2
17:00 - 结束今天的工作
```

**验证点**:
- [ ] 每个时间点的操作都正确执行
- [ ] 所有文件正确创建和更新
- [ ] 最终归档和报告完整

---

### TC-C02: 复杂任务管理
**测试目标**: 验证多任务、跨天的复杂管理

**场景**:
- Day 1: 3 个任务,完成 1 个,进行中 2 个
- Day 2: 延续 2 个,完成 1 个,新增 1 个
- Day 3: 延续 1 个,完成,新增 2 个

**验证点**:
- [ ] 跨天任务延续正确
- [ ] 任务状态追踪准确
- [ ] 最终所有任务归档
